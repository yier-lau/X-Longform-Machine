# 草莽长文 / X Longform Machine

一个用于生成、诊断、重写和发布 X 长文的 Agent Skill。

它不是“爆款提示词模板”，而是一条受控的长文生产线：从选题判断、文章骨架、初稿、风格重写、AI 味检查、利他性检查，到发布包、短帖、评论区补充和封面提示词。

## 适合谁

- 想把零碎想法写成 X 长文的人。
- 想把自己的写作流程封装成 Skill / SOP 的人。
- 想让 AI 写得更像自己，但又不想丢掉判断权的人。
- 想把长文副产物沉淀成短帖、评论区补充、封面提示词等资产的人。

## 核心特点

- **人工审核节点**：选题、骨架、风格重写、发布包、封面提示词等关键节点默认需要用户确认。
- **分享感优先**：默认从“我刚做了什么、我发现了什么、我怎么改”的角度写，不站在读者头上说教。
- **利他性检查**：发布前会检查文章是否对读者有用，是否值得收藏，而不只看标题和传播性。
- **去 AI 味**：检查过度完整、过度顺滑、空泛排比、泛化口号等 AI 写作痕迹。
- **发布资产化**：发布包可以拆出短帖、评论区补充、后续选题和封面提示词。
- **封面提示词**：默认输出适合 X 长文封面的 `5:2` 概念海报提示词。

## 安装

如果你的 Agent 工具支持从 GitHub 安装 Skill，可以使用仓库地址：

```bash
npx skills add yier-lau/X-Longform-Machine
```

也可以手动复制 `skills/x-longform-machine` 到你的本地 skills 目录。

## 基本用法

你可以这样调用：

```text
用草莽长文，把这个想法写成一篇 X 长文。
```

```text
把这篇草稿改得更像我，保留我的语气，不要写成普通 AI 文案。
```

```text
给这篇文章生成发布包，包括短帖、评论区补充、下一篇选题和封面提示词。
```

## 可选个人语料库

这个 Skill 可以独立使用。

如果你有自己的个人表达资产库，可以配置一个 `PERSONAL_EXPRESSION_ROOT`，里面可以包含：

```text
INDEX.md
08_资产库/索引.md
04_公开安全素材库
05_写作风格库
01_选题库
02_金句库
03_故事库
16_问题库
```

没有个人语料库也可以用，只是“像你”的程度会取决于你在当前对话里提供的材料。

## dbskill 依赖说明

本 Skill **不强制依赖** dbskill。

它内置了简化版的目标清晰、内容诊断、开头优化、AI 味检查和状态管理逻辑。

如果你已经安装 dontbesilent 的 dbskill，可以把它作为增强模块使用：

- `dbs-goal`：目标清晰化
- `dbs-deconstruct`：概念拆解
- `dbs-content`：内容诊断
- `dbs-hook`：开头优化
- `dbs-ai-check`：AI 写作特征识别

没有安装 dbskill 时，本 Skill 仍然可以正常运行。

## 公开边界

这个仓库只包含 Skill 规则和通用流程，不包含任何个人语料、聊天记录、日记、发布包、真实项目资料或私密素材。

使用时请注意：

- 不要把私密日记原文直接公开。
- 不要把联系方式、账号 ID、地址、他人隐私写进发布内容。
- 私密材料只能在用户明确允许时作为风格参考，并且必须脱敏。
- 公开版 Skill 应该帮助你保留判断权，而不是替你决定你是谁。

## 推荐流程

```text
选题判断
  ↓
文章骨架
  ↓
初稿
  ↓
风格重写
  ↓
内容诊断
  ↓
AI 味检查
  ↓
利他性 / 收藏价值检查
  ↓
发布精修
  ↓
发布包
  ↓
短帖 / 评论区补充 / 封面提示词
```

## 一句话

草莽长文不是让 AI 替你写成另一个人。

它是让你把自己的判断、风格、边界和流程拆出来，再让 AI 按你的方式干活。
